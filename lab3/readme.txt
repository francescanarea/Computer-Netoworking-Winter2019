File Transfer v2-- Stop and Wait for Unrealiable Channel

Messages are sent one at a time, and each message needs to be acknowledged when received, before a new message can be sent. TFv2implements basically the protocol rdt2.2 presented in the text book.

Use:
1. begin server with port
2. edit input.txt file with desired data to transfer
3. run the client with the server's port
